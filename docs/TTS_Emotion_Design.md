## 一、核心设计

### 1.1 工作流程

```
LLM 回复格式：
<emo>情感</emo>实际回复内容

示例：
<emo>高兴</emo>今天天气真不错！
```

```
┌─────────────────────────────────────────────────────────┐
│                    前端接收 LLM 流式响应                       │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│              情感标签解析模块                                │
│  状态机：TEXT → TAG_OPEN → TAG_CONTENT → TAG_CLOSE → TEXT │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  提取结果：                                              │
│  - emotion: "高兴" (用于 TTS)                            │
│  - text: "今天天气真不错！" (用于显示和 TTS)                │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  TTS 调用：使用解析出的情感播放                           │
└─────────────────────────────────────────────────────────┘
```

## 二、情感标签解析

### 2.1 状态机

```
状态转移：

     ┌─────────┐  '<'   ┌───────────┐  'emo'   ┌────────────┐  '>'   ┌─────────┐  其他字符   ┌─────────┐
TEXT │────────→│ TAG_OPEN │────────→│TAG_VERIFY│────────→│CONTENT │────────→│ TEXT    │
     └─────────┘         └───────────┘           └────────────┘         └─────────┘
                              ↑                                                      │
                              │  不是 'emo'                                            │
                              └───────────────────────────────────────────────────────┘

     ┌─────────┐  '<'   ┌───────────┐  '/'   ┌───────────┐  '>'   ┌─────────┐
TAG_CLOSE │────────→│ SLASH1 │────────→│SLASH2 │────────→│ CHECK │──成功──→│ READY  │
         └─────────┘         └───────────┘         └───────────┘         └─────────┘
```

### 2.2 解析状态

| 状态 | 说明 |
|------|------|
| `TEXT` | 普通文本模式，累积正文 |
| `TAG_OPEN` | 刚遇到 `<`，等待 `emo>` |
| `TAG_VERIFY` | 验证是否为 `<emo>` |
| `TAG_CLOSE` | 遇到 `</`，等待 `emo>` |
| `READY` | 标签解析完成，提取情感成功 |
| `PARSE_DONE` | 完成，可输出 |

### 2.5 解析失败处理

| 情况 | 处理 |
|------|------|
| 标签不在6个情感中 | 默认"高兴" |
| 标签为空 | 默认"高兴" |
| 格式错误 | 忽略标签，当普通文本 |
| 大小写不匹配 | 统一转小写后匹配 |

## 三、提示词设计

### 3.1 提示词内容

```markdown
你的所有回复需要在开头包含情感标签，格式如下：
<emo>情感</emo>

可选情感（必须从中选择一个最符合当前回复内容的）：
- 高兴
- 伤心
- 生气
- 害怕
- 平静
- 嫌弃

注意：
1. 标签必须在回复最开头
2. 标签内的文字只能是上述6个情感之一，不要添加其他符号或文字
3. 如果无法判断或内容不适合任何情感，默认使用"高兴"
4. 标签单独作为一个chunk输出，不要与其他内容混合
```

### 3.2 自动追加机制

| 配置项 | 说明 |
|--------|------|
| 开启条件 | TTS 功能启用时自动追加 |
| 追加位置 | system_prompt 末尾 |
| 影响范围 | 仅当前角色的对话 |

## 四、流式处理

### 4.1 前端流式处理流程

```
LLM stream chunks
     ↓
累积到缓冲区
     ↓
状态机解析
     ↓
遇到 <emo> → 进入标签模式
     ↓
提取情感 → 缓存情感
     ↓
遇到 </emo> → 退出标签模式
     ↓
后续文本 → 触发 TTS (使用缓存情感)
```

### 4.2 示例时序

```
时间  LLM 输出          前端处理              状态
─────────────────────────────────────────────────
T1    "<"             追加到 buffer         TEXT
T2    "<emo"           追加到 buffer         TAG_OPEN
T3    ">"              追加到 buffer         TAG_VERIFY ✓
T4    "高兴"           追加到 buffer         CONTENT
T5    "</emo>"         提取"高兴"            TAG_CLOSE ✓
T6    "今"             触发 TTS + 显示       TEXT
T7    "天"             触发 TTS + 显示
T8    "天"             触发 TTS + 显示
...
```

### 4.3 情感与 TTS 时序

```
情感解析完成（遇到 </emo>）
     ↓
缓存当前情感
     ↓
后续每个 chunk 触发 TTS 时使用缓存的情感
     ↓
直到新的情感标签被解析（重置缓存）
```

## 五、非流式处理

### 5.1 处理流程

```
LLM 返回完整响应
     ↓
解析 <emo>...</emo>
     ↓
提取情感 + 移除标签
     ↓
发送给 TTS
```

### 5.2 示例

```
输入：<emo>高兴</emo>今天天气真不错！
输出：今天天气真不错！ (情感: 高兴.wav)
```

## 十、聊天记录不记录情感标签

### 10.1 设计原则

```
LLM 输出：<emo>高兴</emo>今天天气真不错！

存储/显示：今天天气真不错！✅
TTS：高兴.wav ✅
```

**情感标签只用于 TTS，不存储、不显示、不进入记忆系统。**

### 10.2 状态机返回值

```typescript
interface ParseResult {
  emotion: string   // 解析出的情感（用于 TTS）
  text: string     // 纯净的正文（无标签，用于显示/存储）
  ready: boolean   // 情感是否解析完成
}
```

### 10.3 处理位置

| 模块 | 处理 |
|------|------|
| 流式前端 | 解析标签后，消息 content 只追加 text（无标签） |
| 聊天记录 | 只保存 ParseResult.text |
| 记忆系统 | 只保存 ParseResult.text |
| TTS | 使用 ParseResult.emotion |

### 10.4 示例

**流式处理：**
```
输入: "<emo>高兴</emo>今天天气真不错！"
解析: { emotion: "高兴", text: "今天天气真不错！", ready: true }

消息 content += "今天天气真不错！"  // 无标签
TTS(text, "高兴")              // 用情感播放
记忆 += "今天天气真不错！"      // 无标签
```

### 10.5 代码设计

```typescript
function processChunk(chunk: string, state: ParserState): { state: ParserState, text: string, emotion: string | null } {
  // 状态机解析...
  // 只返回纯净文本用于存储
  return { state, text: pureText, emotion: currentEmotion }
}

// 使用
const { text, emotion } = processChunk(chunk, parserState)
if (text) {
  // 更新消息（显示）
  updateMessage(text)
  // 触发 TTS（使用情感）
  if (emotion) {
    ttsStore.speakWithEmotion(text, emotion)
  }
  // 存储到记忆（无情感标签）
  memoryStore.append(text)
}
```
