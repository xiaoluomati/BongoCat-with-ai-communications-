### 3.1 LLM 对话

- **用例编号**: TC-101
- **前置**: 应用已启动，LLM 已配置
- **步骤**:
  1. 输入 "你好，今天天气怎么样？"
  2. 点击发送
  3. 等待 LLM 回复完成
- **预期**: LLM 返回完整文字回复

#### TC-102: 流式对话

- **用例编号**: TC-102
- **前置**: LLM 已配置

  2. 点击发送
  3. 观察回复是否逐字/逐句出现
- **预期**: 回复内容流式显示（逐字或逐句出现）

#### TC-103: 切换流式/非流式

- **用例编号**: TC-103
- **前置**: -
- **步骤**:
  1. 在设置中开启流式模式，发送消息观察流式效果
  2. 关闭流式模式，发送消息观察非流式效果
- **预期**: 两种模式均正常工作

### 3.2 TTS 语音合成

#### TC-201: 基础 TTS 播放

- **用例编号**: TC-201
- **前置**: TTS 已启用，IndexTTS 服务正常运行
- **步骤**:
  1. 在聊天框输入 "测试语音"
  2. 发送消息
  3. 观察音频是否播放
- **预期**: TTS 正常播放，无爆音、无明显截断

#### TC-202: 指定情感播放

- **用例编号**: TC-202
- **前置**: TTS 已启用，IndexTTS 服务正常运行
- **步骤**:
  1. 输入 `<emo>平静</emo>今天天气真好`
  2. 发送消息
- **预期**: TTS 以"平静"情感朗读内容

#### TC-203: 流式 TTS（边说边播）

- **用例编号**: TC-203
- **前置**: TTS 已启用，流式模式已开启，IndexTTS 正常
- **步骤**:
  1. 输入 "一二三四五，上山打老虎。老虎打不到，打到一只小老鼠。老鼠老鼠你真小，跑到山上吃青草。"
  2. 发送消息
  3. 记录从发送到首次音频播放的时间
  4. 观察句子之间是否依次触发 TTS
- **预期**: 首次音频延迟 < 3秒，每句完整后依次触发 TTS

#### TC-204: 播放控制

- **用例编号**: TC-204
- **前置**: TTS 正在播放
- **步骤**:
  1. 发送一条稍长的消息触发 TTS 播放
  2. 点击暂停按钮
  3. 点击继续按钮
  4. 点击停止按钮
  5. 点击跳过按钮（跳到下一条）
- **预期**:
  - 暂停: 音频暂停
  - 继续: 从暂停位置恢复
  - 停止: 音频停止，清空队列
  - 跳过: 停止当前，播放下一个

#### TC-205: TTS 服务不可用

- **用例编号**: TC-205
- **前置**: IndexTTS 服务未运行
- **步骤**:
  1. 停止 IndexTTS 服务
  2. 发送消息触发 TTS
- **预期**: 显示错误提示，不崩溃，可继续其他操作

### 3.3 情感自动识别（Emotion Auto）

#### TC-301: Emotion Auto 开启/关闭

- **用例编号**: TC-301
- **前置**: TTS 已启用，IndexTTS 正常
- **步骤**:
  1. **关闭 Emotion Auto**: 在设置中关闭"情感自动识别"
  2. 发送 "我今天很开心！"
  3. 观察回复中是否包含 `<emo>` 标签（界面上应被过滤，不显示给用户）
  4. **开启 Emotion Auto**: 打开"情感自动识别"
  5. 发送 "我今天很开心！"
  6. 观察回复是否使用对应情感朗读
- **预期**:
  - 关闭时: TTS 使用默认/配置的音色，不带情感参数
  - 开启时: TTS 根据 LLM 回复的情感标签使用对应情感朗读

#### TC-302: 情感切换

- **用例编号**: TC-302
- **前置**: Emotion Auto 已开启
- **步骤**:
  1. 发送 "我今天非常开心！"
  2. 记录当前情感（TTS 以高兴/开心情感播放）
  3. 发送 "我现在很悲伤..."
  4. 观察 TTS 是否切换到伤心/平静情感
- **预期**: TTS 情感随消息内容变化

#### TC-303: 情感权重（emo_weight）

- **用例编号**: TC-303
- **前置**: Emotion Auto 已开启，IndexTTS 正常
- **步骤**:
  1. 设置 emo_weight = 0.3（低）
  2. 发送带情感的消息，观察情感效果
  3. 设置 emo_weight = 1.0（高）
  4. 发送同样的消息
- **预期**: emo_weight 影响情感表现的强弱

### 3.4 用户画像

#### TC-401: 手动刷新画像

- **用例编号**: TC-401
- **前置**: 有对话记录，profile 页面可访问
- **步骤**:
  1. 在综合功能页打开"用户画像"
  2. 点击"刷新画像"按钮
  3. 等待 LLM 分析完成
- **预期**:
  - 画像内容更新（性格、偏好、重要日期、互动、回忆）
  - "已整理轮数"字段更新为当前对话轮数

#### TC-402: 自动更新画像（满50轮）

- **用例编号**: TC-402
- **前置**: 对话记录接近 50 轮
- **步骤**:
  1. 确认当前对话轮数（看"已整理轮数"）
  2. 继续对话直到满 50 轮新消息
  3. 发送第 50 条新消息后等待
- **预期**: 满 50 轮后自动触发画像更新（无需手动点击）

#### TC-403: 画像已是最新

- **用例编号**: TC-403
- **前置**: 画像已是最新（对话轮数 == 已整理轮数）
- **步骤**:
  1. 确认"已整理轮数" == 当前对话总轮数
  2. 点击"刷新画像"
- **预期**: 提示"画像已是最新，无需更新"

#### TC-404: 无对话记录时刷新

- **用例编号**: TC-404
- **前置**: 没有任何对话记录
- **步骤**:
  1. 确认记忆管理中无任何聊天记录
  2. 点击"刷新画像"
- **预期**: 提示"暂无对话数据"

### 3.6 配置 UI

#### TC-601: TTS 基础配置

- **用例编号**: TC-601
- **前置**: -
- **步骤**:
  1. 打开 TTS 设置
  2. 修改: 启用/禁用开关、音量（0-100）、语速（0.5-2.0）
  3. 保存配置
  4. 重启应用或切换页面后回来
  5. 验证配置已保持
- **预期**: 配置正确保存，重启后保持

#### TC-602: 添加自定义音色

- **用例编号**: TC-602
- **前置**: IndexTTS 服务正常运行，可获取音色列表
- **步骤**:
  1. 在 TTS 设置中点击"获取服务器音色"
  2. 从列表中选择一个音色添加
  3. 设置该音色的语速（speed）
  4. 保存并设为默认音色
  5. 发送消息验证使用了新音色
- **预期**: 音色添加成功，发送消息时使用新音色

#### TC-603: 流式 TTS 参数

- **用例编号**: TC-603
- **前置**: 流式模式已开启
- **步骤**:
  1. 设置 stream_trigger_threshold（触发阈值）
  2. 设置 stream_max_buffer（最大缓冲）
  3. 设置 fade_duration（淡出时长）
  4. 发送长消息观察效果
- **预期**: 参数变化影响流式播放的行为（触发时机、缓冲、流畅度）
